How it works

Five agents, one unbroken pipeline.

Each agent produces a markdown artefact that is the next agent's sole input. The Manager runs quality gates between every handoff and maintains a cycle log. This site is the third artefact; the others live in the repository.

1 Researcher
Dr. Aiko Tanaka

Analyses the data. Produces 01-research-brief.md: four at-risk segments, five testable hypotheses.

2 Designer
Leo Marín

Designs the journey. Produces 02-design-spec.md: four flyer archetypes, Octalysis-anchored.

3 Maker
Priya Shah

Builds the prototype. Produces this docs/ site, no framework, no build step.

4 Communicator
Dara Okafor

Writes the GTM pack. Produces 04-gtm-pack.md: positioning, three emails, press release.

5 Manager
Claire Dubois

Synthesises. Produces 05-executive-summary.md: KPIs, ethics, rollout, rubric self-assessment.

Built for trust

Loyalty done the way it should be.

Our previous model assumed loyalty looked like cancellations. It does not. Loyalty looks like a flyer quietly choosing another carrier for the next trip.

We do not chase. We notice. Every contact opens with the reason it was sent. Every "surprise" is a real benefit, already loaded to the account. Every risk score is reviewed by a Customer Success Manager before a flyer ever hears from us.

Octalysis Framework

Seven of eight Core Drives, one deliberately absent.

The flyer-facing view explicitly tags every gamification element with its Octalysis Core Drive via a data-cd attribute. CD6 Scarcity (countdown timers, fake seat counts) is deliberately omitted from the pilot.
